# Tondix CRM — Design Spec

**Date :** 2026-04-09  
**Secteur :** Vente et entretien de tondeuses (B2B)  
**Stack :** React 19 + TypeScript + React Admin (ra-core) + Supabase  
**Identité visuelle :** Vert (#16a34a) et Jaune (#ca8a04)

---

## Contexte

Tondix est un vendeur de tondeuses B2B. Ce CRM adapte Atomic CRM pour gérer :
- Le catalogue produits (tondeuses) et services (entretiens)
- La vente de produits et services via les deals
- Le suivi des contrats d'entretien par entreprise cliente
- Les relances d'entretiens à partir du dashboard

---

## 1. Modèle de données

### Nouvelles tables

#### `products` — Catalogue tondeuses
```sql
id            bigint generated by default as identity primary key
name          text not null
description   text
price         numeric not null
active        boolean not null default true
created_at    timestamp with time zone not null default now()
```

#### `services` — Catalogue entretiens
```sql
id                  bigint generated by default as identity primary key
name                text not null
type                text                          -- complet, intermédiaire, rapide…
periodicity_months  integer not null              -- périodicité en mois
price               numeric not null
description         text
active              boolean not null default true
created_at          timestamp with time zone not null default now()
```

#### `deal_products` — Liaison deal ↔ tondeuses
```sql
id          bigint generated by default as identity primary key
deal_id     bigint not null references public.deals(id) on delete cascade
product_id  bigint not null references public.products(id)
quantity    integer not null default 1
unit_price  numeric not null                      -- snapshot du prix au moment du deal
```

#### `deal_services` — Liaison deal ↔ entretiens
```sql
id          bigint generated by default as identity primary key
deal_id     bigint not null references public.deals(id) on delete cascade
service_id  bigint not null references public.services(id)
quantity    integer not null default 1
unit_price  numeric not null                      -- snapshot du prix au moment du deal
```

#### `maintenance_contracts` — Contrats d'entretien
```sql
id          bigint generated by default as identity primary key
company_id  bigint not null references public.companies(id) on delete cascade
service_id  bigint not null references public.services(id)
deal_id     bigint references public.deals(id)
start_date  date not null
end_date    date not null                         -- start_date + periodicity_months
status      text not null default 'active'        -- active | expired | cancelled
created_at  timestamp with time zone not null default now()
```

**Création automatique :** quand un deal passe au stage `won`, un `maintenance_contract` est créé pour chaque `deal_service` associé. `start_date` = date du jour où le deal bascule en `won` (pas `expected_closing_date`), `end_date` = start_date + `periodicity_months` mois.

**Note sur le statut :** le champ `status` est positionnel (défaut `active`). Il n'est pas mis à jour automatiquement. Les requêtes de filtrage et la vue `companies_summary` utilisent `end_date > now()` pour déterminer si un contrat est réellement actif (et non le champ `status`).

### Vue Supabase étendue

`companies_summary` étendue avec deux champs calculés :
- `has_equipment` (bool) — a au moins un `deal_product` via ses deals
- `has_maintenance` (bool) — a au moins un `maintenance_contract` avec `end_date > now()`

## 2. Navigation

Ajout d'une section **Catalogue** dans la sidebar avec deux sous-entrées :
```
Catalogue
  ↳ Tondeuses     → /catalogue/products
  ↳ Entretiens    → /catalogue/services
```

## 3. Pages Catalogue

### 3.1 Tondeuses (`/catalogue/products`)

**Liste :**
- Colonnes : Nom, Prix, Statut (Actif / Inactif), Actions (modifier, supprimer)
- Filtres rapides : Toutes / Actives / Inactives
- Recherche texte sur le nom
- Bouton "+ Ajouter une tondeuse"

**Formulaire (création & édition) :**
- Nom * (text)
- Description (textarea)
- Prix * (number)
- Actif / Inactif (toggle)

### 3.2 Entretiens (`/catalogue/services`)

**Liste :**
- Colonnes : Nom, Type, Périodicité, Prix, Statut, Actions
- Filtres rapides : Toutes / Actives / Inactives
- Recherche texte
- Bouton "+ Ajouter un entretien"

**Formulaire :**
- Nom * (text)
- Type (text : complet, intermédiaire, contrôle rapide…)
- Périodicité * (number, en mois)
- Prix * (number)
- Description (textarea)
- Actif / Inactif (toggle)

## 4. Modification du formulaire Deal

Deux nouvelles sections ajoutées sous les champs existants du deal.

### Section Tondeuses (couleur verte)
- Bouton "+ Ajouter une tondeuse"
- Chaque ligne : `ReferenceInput` → `products` (uniquement actifs), quantité (int), prix unitaire (pré-rempli, modifiable)
- Bouton de suppression par ligne
- Lien "Gérer le catalogue →" ouvrant `/catalogue/products` en cas de produit manquant

### Section Entretiens (couleur jaune)
- Bouton "+ Ajouter un entretien"
- Chaque ligne : `ReferenceInput` → `services` (uniquement actifs), quantité (int), prix unitaire (pré-rempli, modifiable)
- Bouton de suppression par ligne
- Lien "Gérer le catalogue →" ouvrant `/catalogue/services`

### Note informationnelle
Bandeau bleu dans le formulaire : *"Quand ce deal passe en 'Gagné', un contrat d'entretien sera automatiquement créé pour chaque entretien sélectionné."*

---

## 5. Modification de la fiche Entreprise

### 5.1 Corps principal — nouvelles sections

Sous les informations générales existantes, deux sections colorées :

#### Section Équipements (fond vert clair, bordure verte)
- Titre : "🌿 Équipements achetés" + badge compteur
- Tableau : Produit, Quantité, Deal (lien cliquable), Date
- Données issues de `deal_products` → `deals` (company_id = cette entreprise)

#### Section Contrats d'entretien (fond jaune clair, bordure jaune)
- Titre : "🔧 Contrats d'entretien" + badge "X actif(s)"
- Tableau : Service, Début, Fin, Statut (Actif vert / Expiré rouge)
- Données issues de `maintenance_contracts` (company_id = cette entreprise)

### 5.2 Filtres sur la liste des entreprises

Deux nouveaux filtres dans la barre de filtres de `/companies` :
- **🌿 Avec équipement** — filtre sur `has_equipment = true`
- **🔧 Avec entretien actif** — filtre sur `has_maintenance = true`

---

## 6. Widget Dashboard — Entretiens à relancer

Nouveau widget pleine largeur en bas du dashboard.

**En-tête :** dégradé vert → jaune, titre "🔧 Entretiens à relancer", badges compteurs.

**Sous-section "Expire bientôt" (orange) :**
- Contrats avec `end_date` entre aujourd'hui et aujourd'hui + 30 jours
- Colonnes : Entreprise, Service, "Dans X jours", Date d'expiration
- Chaque ligne est un lien cliquable vers la fiche de l'entreprise

**Sous-section "Récemment expirés" (rouge) :**
- Contrats avec `end_date` entre aujourd'hui - 60 jours et aujourd'hui
- Même structure que ci-dessus, affichage "Expiré il y a Xj"
- Chaque ligne est un lien cliquable vers la fiche de l'entreprise

**Pied :** lien "Voir toutes les entreprises avec entretien →" pointant vers `/companies` filtré sur `has_maintenance = true`.
